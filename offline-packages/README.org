#+title: Offline Emacs Toolkit
#+author: James Dyer
#+options: toc:2 num:nil
#+startup: content

A self-contained toolkit for bringing up my Emacs configuration on machines with no network access. Builds per-distro, per-version ELPA mirrors on an online machine and packages everything into a single tarball that can be dropped on an offline target.

* Overview

The workflow has three pieces:

1. =build-mirror.sh= — run on an /online/ machine that matches the target's Emacs version and OS. Builds an ELPA mirror tarball containing every currently-installed package.
2. =build-toolkit.sh= — run on any /online/ machine with access to this repo. Picks one of the mirrors built in step 1 and packages it with the shared =init.el=, the =Emacs-vanilla= + =Emacs-DIYer= literate configs, an installer, and (optionally) the GNU Emacs source tarball.
3. =setup.sh= — lives inside each built toolkit, run on the /offline/ target. Drops everything into =~/.emacs.d/= and extracts the mirror into =~/=.

The =init.el= auto-detects =~/elpa-mirror-emacs-*/= at startup and points =package-archives= at the local mirror when =my/offline-packages= is =t=.  No edits required after install.

* Directory layout

#+begin_example
offline-packages/
├── build-mirror.sh         ← run on online machine matching target
├── build-toolkit.sh        ← assemble a deployable toolkit
├── init.el                 ← shared, offline-aware init for all targets
├── README.org              ← this file
└── emacs-<VER>/<OS>/<ARCH>/
    └── elpa-mirror-emacs-<VER>-<OS>-<ARCH>-<STAMP>-<N>pkgs.tar.gz
#+end_example

Each target directory holds exactly one (or many, newest wins) ELPA mirror tarball built by =build-mirror.sh=. The =emacs-<VER>=, =<OS>= (slug from =/etc/os-release=), and =<ARCH>= (=uname -m=) components come from the machine on which the mirror was built.

Current targets:

#+begin_src sh :results output :exports results
./build-toolkit.sh --list
#+end_src

* Building an ELPA mirror (step 1)

Run on the online machine. It must match the offline target's Emacs version and OS as closely as possible, because some packages ship native-compiled =.eln= artefacts tied to the build environment.

#+begin_src sh
cd ~/.emacs.d/offline-packages
./build-mirror.sh               # writes under $HOME by default
./build-mirror.sh /path/to/out  # or pick an explicit output directory
#+end_src

Output tarball layout:

#+begin_example
<OUT_DIR>/emacs-<VER>/<OS>/<ARCH>/elpa-mirror-emacs-<VER>-<OS>-<ARCH>-<STAMP>-<N>pkgs.tar.gz
#+end_example

Inside the tarball:

| File                 | Purpose                                                   |
|----------------------+-----------------------------------------------------------|
| =archive-contents=   | ELPA index Emacs uses for =package-install=               |
| package =.tar=/=.el= | One artefact per installed package                        |
| =MANIFEST.txt=       | Build provenance (host, kernel, Emacs version, timestamp) |
| =PACKAGES.txt=       | Sorted =package  version  summary= listing                |

Move the output into this repo under =offline-packages/emacs-<VER>/<OS>/<ARCH>/= and commit, so =build-toolkit.sh= can find it.

* Building a deployable toolkit (step 2)

Interactive:

#+begin_src sh
cd ~/.emacs.d/offline-packages
./build-toolkit.sh
#+end_src

Picks a target interactively (via =fzf= if available, otherwise a numbered menu) and writes =$HOME/emacs-offline-toolkit-...tar.xz=.

Non-interactive:

#+begin_src sh
./build-toolkit.sh --target emacs-27.2/opensuse-leap-15.4/x86_64
./build-toolkit.sh --target emacs-30.2/garuda/x86_64 --out-dir /tmp
#+end_src

With a bundled GNU Emacs source tarball for rebuild on the target:

#+begin_src sh
./build-toolkit.sh -t emacs-27.2/opensuse-leap-15.4/x86_64 -s 27.2
#+end_src

Flags:

| Flag                    | Purpose                                    |
|-------------------------+--------------------------------------------|
| =-t=, =--target SPEC=   | =emacs-VER/OS/ARCH= spec; skip picker      |
| =-o=, =--out-dir DIR=   | Output directory (default: =$HOME=)        |
| =-s=, =--with-source VER= | Download + bundle =emacs-VER.tar.xz=     |
| =-l=, =--list=          | List available targets and exit            |
| =-h=, =--help=          | Show help                                  |

** What goes into the toolkit

#+begin_example
emacs-offline-toolkit-<VER>-<OS>-<ARCH>-<STAMP>/
├── MANIFEST.txt            ← build provenance + file inventory
├── README.org              ← this document
├── init.el                 ← shared offline-aware init
├── early-init.el           ← if present in ~/.emacs.d/
├── setup.sh                ← installer for the offline machine
├── elpa-mirror-*.tar.gz    ← the selected ELPA mirror
├── Emacs-vanilla/          ← literate config (READMEs, init.el, scripts)
├── Emacs-DIYer/            ← literate config (READMEs, init.el, dictionary)
└── emacs-<VER>.tar.xz      ← optional GNU Emacs source
#+end_example

Excluded from =Emacs-vanilla/= and =Emacs-DIYer/= to keep the toolkit small:

- =.git=, =.gitignore=, =.gitmodules=, =.github=, =.claude=
- =*.elc=, =*~=, =.#*=, =#*#=
- =eln-cache=, =auto-save-list=

** Size

Compression is =xz -9e= (extreme preset). Typical sizes:

| Target           | Mirror | Configs | Total (no source) | Total (+source) |
|------------------+--------+---------+-------------------+-----------------|
| Emacs 27.2 (14 pkgs) | ~600 KB | ~700 KB | ~1–2 MB          | ~55–65 MB       |
| Emacs 30.2 (28 pkgs) | ~1.7 MB | ~700 KB | ~2–3 MB          | ~55–65 MB       |

The Emacs source is already =xz=-compressed, so the outer =xz= adds minor overhead but simplifies the distribution to a single archive.

* Deploying on the offline target (step 3)

Copy the =.tar.xz= to the target by whatever means (USB stick, air-gapped file transfer, etc.), then:

#+begin_src sh
tar -xJf emacs-offline-toolkit-*.tar.xz
cd emacs-offline-toolkit-*/
./setup.sh
#+end_src

=setup.sh= will:

1. Back up any existing =~/.emacs.d/init.el=, =early-init.el=, =Emacs-vanilla/=, and =Emacs-DIYer/= with a timestamped =.pre-toolkit-<STAMP>= suffix.
2. Extract the ELPA mirror tarball into =~/=, creating =~/elpa-mirror-emacs-<VER>-<OS>-<ARCH>-<STAMP>-<N>pkgs/=.
3. Install the literate config directories under =~/.emacs.d/=.
4. Install =init.el= (and =early-init.el= if bundled) under =~/.emacs.d/=.
5. Report instructions for building the bundled Emacs source, if present.

Launch Emacs and check =*Messages*=. You should see:

#+begin_example
my/offline-packages=t  dir=/home/you/elpa-mirror-emacs-...
#+end_example

If =dir=<none found>= appears, the ELPA mirror wasn't extracted into =$HOME= or the pattern =~/elpa-mirror-emacs-*= didn't match — see Troubleshooting.

* How the online/offline toggle works

The shared =init.el= ([[file:init.el]]) exposes one variable:

#+begin_src emacs-lisp
(setq my/offline-packages t)       ; flip to nil for online archives
(setq my/offline-packages-dir
      (car (sort (file-expand-wildcards
                  (expand-file-name "~/elpa-mirror-emacs-*"))
                 #'string>)))
#+end_src

- =t= + a discovered mirror dir → =package-archives= is set to the local mirror only.
- =nil= (or no mirror found) → =package-archives= falls back to MELPA / ELPA / NonGNU ELPA / Org ELPA over the network.

The mirror directory is detected automatically by globbing =~/elpa-mirror-emacs-*=; newest wins. No edits needed when a new mirror is installed.

All =require= calls for optional packages are wrapped in =(require 'foo nil t)= so a missing package logs to =*Messages*= rather than aborting init.  =custom-set-variables=, =custom-set-faces=, and the trailing =load-theme= are wrapped in =condition-case= for the same reason — useful when running a pared-down mirror that lacks certain themes.

* Rebuilding Emacs from the bundled source (optional)

When =--with-source VER= was used at toolkit build time, an =emacs-<VER>.tar.xz= is bundled. To build:

#+begin_src sh
tar -xf emacs-<VER>.tar.xz -C /tmp
cd /tmp/emacs-<VER>
./configure --prefix="$HOME/.local"
make -j$(nproc)
make install
export PATH="$HOME/.local/bin:$PATH"
#+end_src

Dependencies the configure step will typically want installed from the distro's /own/ offline repos:

- =libgtk-3-dev= or =libncurses-dev= depending on whether you want GUI or TUI
- =libgnutls28-dev=, =libxml2-dev=, =libjpeg-dev=, =libpng-dev=, =libtiff-dev=
- =libgif-dev=, =libjansson-dev= (for native JSON), =libtree-sitter-dev=

These are outside the toolkit — each distro bundles them differently.

* Troubleshooting

** Package X skipped at startup

Look in =*Messages*= for lines like:

#+begin_example
my/ensure-package: skipping ef-themes ((error . "..."))
#+end_example

Reasons this normally happens offline:
- Package requires a newer Emacs than the target (=ef-themes= needs 28.1+).
- Package wasn't in the online machine's =package-alist= when the mirror was built — rebuild the mirror with the package installed first.

** my/offline-packages-dir=<none found>

The ELPA mirror didn't get extracted into =$HOME=. Run:

#+begin_src sh
ls -d ~/elpa-mirror-emacs-*
#+end_src

If there's no match, re-run =setup.sh= from inside the extracted toolkit directory, or extract the mirror manually:

#+begin_src sh
tar -xzf ~/.../elpa-mirror-emacs-*.tar.gz -C "$HOME"
#+end_src

** Theme file not found

The toolkit bundles whatever was on the online machine at mirror-build time.  If a theme isn't present the =load-theme= calls log a message and init continues. Either pick a different theme, or rebuild the mirror after installing the theme.

** Mixed Emacs versions

Mirrors are version-locked (the name encodes the Emacs version and the packages' =:defined-in= metadata may reference version-specific files). Do /not/ build a mirror on Emacs 30 and deploy it to Emacs 27.2 — some packages (e.g. =transient=) will be rejected by =package-install=.
